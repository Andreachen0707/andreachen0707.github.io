Leetcode 15. 3Sum

Nov 4, 2018

This problem remind me of Leetcode 5.Longest Palindromic Substring. Similar to that problem, this one also use left and right pointer to do something.

Breaking this problem to 1sum + 2sum will give TLE error, so I have to look at other ways.

Key to this problem is to sort the numbers at the very beginning, so that we won’t have repeat element in the output.

The time complexity of this solution is O(n^2) because approx will be n(n-1). However, this solution in python is extremly slow… I can only be at around 1200ms.

Another interesting fact about this problem is the usage of while. By using while, we can pass all situations that satisfy current condition within one line instead of using if…continue.
